# Высокопроизводительный асинхронный TCP-сервер

Данный проект представляет собой простой, высокопроизводительный и расширяемый TCP-сервер на языке Java, построенный с использованием неблокирующего ввода-вывода Java NIO. Архитектура сервера спроектирована по принципу разделения ответственности (SRP) и полностью развязывает системную сетевую логику и протокол обработки данных.

## Архитектурные особенности

- **Разделение контрактов и реализаций**: Все публичные интерфейсы и сущности расположены на верхнем уровне модулей (core, handler, config), тогда как конкретные реализации скрыты в подпакетах impl и объявлены как final.
- **Client-Worker Affinity**: Каждое новое клиентское подключение распределяется балансировщиком Round-Robin на конкретный рабочий поток EventLoopWorker и регистрируется в его приватном селекторе. Все последующие операции ввода-вывода (чтение, запись в очередь сессии, отправка и отключение) выполняются строго в одном потоке, что исключает накладные расходы на межпоточную синхронизацию памяти и увеличивает процессорный cache hit rate.
- **Многоуровневый Graceful Shutdown**: При остановке сервера акцептор сразу прекращает принимать новые TCP-соединения, а рабочие потоки переходят в режим дренажа, завершая отправку всех находящихся в очередях клиентов байт перед закрытием сокетов. Безопасность процесса контролируется 3-секундным таймаутом.

## Установка и запуск

Для сборки и запуска проекта требуется Java Development Kit (JDK) версии 21.

### Локальный запуск

Сборка проекта и запуск тестов выполняются стандартными командами Gradle Wrapper:

```bash
./gradlew test
```

Для запуска сервера локально выполните:

```bash
./gradlew run
```

По умолчанию сервер считывает конфигурацию из файла ресурсов `server.properties` и запускается на `127.0.0.1:8080`. Для ручной отправки сообщений можно использовать утилиту netcat:

```bash
nc 127.0.0.1 8080
```

### Запуск в Docker

Сервер может быть легко запущен в виде легковесного изолированного Docker-контейнера на базе JRE Alpine.

Сборка образа:

```bash
docker build -t tcp-server:latest .
```

Запуск контейнера с пробросом порта:

```bash
docker run -d -p 8080:8080 --name tcp-server tcp-server:latest
```

## Автоматическая сборка (CI/CD)

В репозитории настроен рабочий процесс GitHub Actions, который при любом пуше в ветку main автоматически выполняет сборку Docker-контейнера и публикует готовый образ в хранилище GitHub Container Registry (GHCR).
